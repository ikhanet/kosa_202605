# PRD — 제품 요구사항 정의서

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| 제품명 | 칸반보드 (Kanban Board) |
| 버전 | v1.2 |
| 작성일 | 2026-05-20 |
| 최종 수정 | 2026-05-20 |
| 작성자 | ikhanet |

Supabase 백엔드 기반의 협업 칸반보드 웹 애플리케이션이다.  
사용자 인증(Google, GitHub, 이메일), 팀 공유, 실시간 동기화, 활동 로그를 지원한다.

---

## 2. 목표

- 사용자가 카드(태스크)를 만들고 3단계 상태(To-Do → In-Progress → Done)로 관리할 수 있다.
- 마우스 드래그 앤 드롭으로 카드를 컬럼 간 이동할 수 있다.
- 외부 의존성 없이 단일 디렉터리에서 동작한다.

---

## 3. 사용자

| 페르소나 | 설명 |
|----------|------|
| 개인 사용자 | 개인 TODO 및 진행 중인 작업을 시각적으로 추적하려는 사람 |
| 학습자 | 칸반 방식으로 학습 진도를 관리하려는 수강생 |
| 팀 사용자 | 보드를 팀원과 공유하고 실시간으로 협업하려는 사람 |

---

## 4. 기능 요구사항

### 4.1 컬럼 구성 (필수)

| ID | 요구사항 |
|----|----------|
| F-01 | 보드는 **To-Do**, **In-Progress**, **Done** 3개 컬럼을 갖는다. |
| F-02 | 각 컬럼에는 제목과 현재 카드 수가 표시된다. |

### 4.2 카드 관리 (필수)

| ID | 요구사항 |
|----|----------|
| F-03 | 각 컬럼 하단의 `+ 카드 추가` 버튼을 클릭하면 인라인 입력창이 열린다. |
| F-04 | 입력창에 텍스트를 입력하고 확인(Enter 또는 버튼)하면 해당 컬럼에 카드가 생성된다. |
| F-05 | 빈 텍스트로는 카드를 생성할 수 없다. |
| F-06 | 카드 우상단 `×` 버튼을 클릭하면 카드가 삭제된다. |

### 4.3 드래그 앤 드롭 (필수)

| ID | 요구사항 |
|----|----------|
| F-07 | 카드를 드래그해서 다른 컬럼으로 이동할 수 있다. |
| F-08 | 드래그 중인 카드는 반투명(opacity 감소)으로 표시된다. |
| F-09 | 드롭 가능한 컬럼에 hover 시 시각적 하이라이트가 표시된다. |
| F-10 | 드롭 후 카드는 해당 컬럼의 카드 목록 맨 아래에 추가된다. |

### 4.4 데이터 유지 (필수)

| ID | 요구사항 |
|----|----------|
| F-11 | 페이지를 새로고침해도 카드 데이터가 유지된다 (Supabase 클라우드 저장). |

### 4.5 사용자 인증 (필수)

| ID | 요구사항 |
|----|----------|
| F-12 | Google / GitHub OAuth 또는 이메일+비밀번호로 로그인할 수 있다. |
| F-13 | 비로그인 상태에서 보드 접근 시 자동으로 로그인 페이지로 이동한다. |
| F-14 | 헤더에 로그인한 사용자의 이메일과 로그아웃 버튼이 표시된다. |

### 4.6 카드 상세 속성 (필수)

| ID | 요구사항 |
|----|----------|
| F-15 | 카드를 클릭하면 상세 모달이 열린다. |
| F-16 | 모달에서 카드 내용, 우선순위(높음/보통/낮음), 마감일, 태그를 편집할 수 있다. |
| F-17 | 카드에는 우선순위 배지, 마감일, 태그가 인라인으로 표시된다. |
| F-18 | 마감일이 오늘보다 이전인 경우 마감일 배지가 빨간색으로 표시된다. |

### 4.7 보드 공유 (필수)

| ID | 요구사항 |
|----|----------|
| F-19 | 보드 소유자는 "공유 링크 복사" 버튼으로 초대 URL을 클립보드에 복사할 수 있다. |
| F-20 | 초대 링크를 클릭한 사용자는 로그인 후 해당 보드에 자동으로 참여된다. |
| F-21 | 멤버는 보드 소유자와 동등한 권한(카드 CRUD)을 가진다. |

### 4.8 활동 로그 (필수)

| ID | 요구사항 |
|----|----------|
| F-22 | 카드 추가/삭제/컬럼 이동 시 활동이 자동으로 기록된다. |
| F-23 | 헤더의 "로그" 버튼을 클릭하면 우측 사이드패널에 활동 이력이 표시된다. |
| F-24 | 활동 로그에는 행위자 이메일, 행동 설명, 시각이 표시된다. |

### 4.9 실시간 협업 (필수)

| ID | 요구사항 |
|----|----------|
| F-25 | 다른 사용자의 보드 변경사항이 실시간으로 반영된다. |
| F-26 | 헤더에 현재 보드를 보고 있는 사용자의 아바타가 표시된다. |

---

## 5. 비기능 요구사항

| ID | 요구사항 |
|----|----------|
| NF-01 | Supabase SDK 외 외부 UI 라이브러리 없이 구현한다. |
| NF-02 | Chrome / Firefox / Edge 최신 버전에서 동작한다. |
| NF-03 | HTML, CSS, JS는 반드시 분리된 파일로 작성한다. |
| NF-04 | 카드 추가/이동/삭제의 DOM 반응은 즉각적이어야 한다. |
| NF-05 | 모든 데이터 접근은 Supabase RLS로 보호된다. |

---

## 6. 범위 외 (Out of Scope)

- 멀티 보드 지원 (사용자당 1개 보드)
- 모바일 터치 드래그 (HTML5 DnD는 데스크톱 전용)
- 카드 댓글 / 첨부 파일
- 역할 구분 (소유자/멤버 동등 권한)
