# PRD — 제품 요구사항 정의서

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| 제품명 | 칸반보드 (Kanban Board) |
| 버전 | v1.0 |
| 작성일 | 2026-05-20 |
| 작성자 | ikhanet |

순수 프론트엔드(HTML/CSS/JS)로 구현하는 칸반보드 웹 애플리케이션이다.  
별도 서버나 데이터베이스 없이 브라우저에서 즉시 동작하며, 업무 또는 학습 태스크를 시각적으로 관리한다.

---

## 2. 목표

- 사용자가 카드(태스크)를 만들고 3단계 상태(To-Do → In-Progress → Done)로 관리할 수 있다.
- 마우스 드래그 앤 드롭으로 카드를 컬럼 간 이동할 수 있다.
- 외부 의존성 없이 단일 디렉터리에서 동작한다.

---

## 3. 사용자

| 페르소나 | 설명 |
|----------|------|
| 개인 사용자 | 개인 TODO 및 진행 중인 작업을 시각적으로 추적하려는 사람 |
| 학습자 | 칸반 방식으로 학습 진도를 관리하려는 수강생 |

---

## 4. 기능 요구사항

### 4.1 컬럼 구성 (필수)

| ID | 요구사항 |
|----|----------|
| F-01 | 보드는 **To-Do**, **In-Progress**, **Done** 3개 컬럼을 갖는다. |
| F-02 | 각 컬럼에는 제목과 현재 카드 수가 표시된다. |

### 4.2 카드 관리 (필수)

| ID | 요구사항 |
|----|----------|
| F-03 | 각 컬럼 하단의 `+ 카드 추가` 버튼을 클릭하면 인라인 입력창이 열린다. |
| F-04 | 입력창에 텍스트를 입력하고 확인(Enter 또는 버튼)하면 해당 컬럼에 카드가 생성된다. |
| F-05 | 빈 텍스트로는 카드를 생성할 수 없다. |
| F-06 | 카드 우상단 `×` 버튼을 클릭하면 카드가 삭제된다. |

### 4.3 드래그 앤 드롭 (필수)

| ID | 요구사항 |
|----|----------|
| F-07 | 카드를 드래그해서 다른 컬럼으로 이동할 수 있다. |
| F-08 | 드래그 중인 카드는 반투명(opacity 감소)으로 표시된다. |
| F-09 | 드롭 가능한 컬럼에 hover 시 시각적 하이라이트가 표시된다. |
| F-10 | 드롭 후 카드는 해당 컬럼의 카드 목록 맨 아래에 추가된다. |

### 4.4 데이터 유지 (선택)

| ID | 요구사항 |
|----|----------|
| F-11 | 페이지를 새로고침해도 카드 데이터가 유지된다 (`localStorage` 활용). |

---

## 5. 비기능 요구사항

| ID | 요구사항 |
|----|----------|
| NF-01 | 외부 라이브러리 없이 순수 HTML/CSS/JS로 구현한다. |
| NF-02 | Chrome / Firefox / Edge 최신 버전에서 동작한다. |
| NF-03 | HTML, CSS, JS는 반드시 분리된 파일로 작성한다. |
| NF-04 | 카드 추가/이동/삭제 응답은 즉각적이어야 한다 (지연 없음). |

---

## 6. 범위 외 (Out of Scope)

- 사용자 인증 / 로그인
- 서버 저장 / 실시간 동기화
- 마감일, 우선순위, 레이블 등 고급 카드 속성
- 멀티 보드 지원
- 모바일 터치 드래그 (HTML5 DnD는 데스크톱 전용)
